ios-driver

IOS automation for native, hybrid and mobile web.

Building from Source

This is a how-to guide on getting everything you need to setup and build different branches. This guide assumes that you have the latest version of Xcode installed.

Installing Maven

You will need the latest version of Maven in order to build and compile. Go to http://maven.apache.org/download.cgi and download the latest version. Unzip this and place it on your desktop. Open a new terminal window and run the following commands: 

                    export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.7.0_45.jdk/Contents/Home <- This could be different depending on installed jdk
                        export M2_HOME=/Users/{YourAccountName}/Desktop/apache-maven-3.1.0 <- This could be different depending on which version of maven you downloaded
                        export PATH=$JAVA_HOME/bin:$PATH
                        export M2=$M2_HOME/bin
                        export PATH=$M2:$PATH

Forking ios-driver

Open a browser window and go to: https://github.com/ios-driver/ios-driver/. Once here (assuming you have a GitHub account) you will need to Fork the branch you want.

Installing GitHub Application

I personally like a GUI so I would recommend using the GitHub Mac application. This is entirely optional but it does make it easy to use. Once you have the application open select "Branches" from the left hand menu. Double click the branch you want and then click the Sync Branch button.

Building from Maven

At this point everything should be setup and installed so that you can begin building the latest branch. In your terminal window, browse to the ios-driver folder. Once in the folder you should run the following commands: 

                            sudo mvn clean install

You should see maven begin building and compileing the ios-driver branch. At the end you should get SUCCESS for all the compilations. If you would prefer to not run the "tests" simply add -DskipTests at the end of the command above.

Running ios-server

It's a good idea to go ahead and give permissions to ios-driver before running. (Do this once) 

                            sudo chmod a+rw /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/SDKs/iPhoneSimulator7.0.sdk/Applications
                            sudo chmod -R a+rw /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/SDKs/iPhoneSimulator7.0.sdk/Applications/MobileSafari.app

At this point in your terminal window drill down to the "/server/target" folder. Once inside you should be able to load ios-server and perform your testing on the latest branch. 

                            java -jar ios-server-0.6.5-jar-with-dependencies.jar -port 5555 -simulators

If you see any errors complaining about ios-driver.log run the following: 

                            sudo chmod -R a+rw /Users/{YourAccountName}/Desktop/ios-driver/server/target
